3과목 : 데이터 분석 기획

3_1 :: 데이터 분석 기획의 이해

3_1_1_1 :: 분석기획 방향성 도출

 - 분석 기획의 특징 :: Math & Statistics / Inforamtion Technology / Domain Knowlede
 - 분석의 대상(Known-Unknown), 분석의 방법(Known-Unknown) 
   - Optimization, Solution, Insight, Discovery
 - 목표시점 :: 과제 중심적 접근, 장기적인 마스터 플랜 방식 --> 서로 융합적으로 하는 것이 중요
 - 분석기술(IT 및 프로그래밍), 도메인 전문성, 의사소통 + 프로젝트관리역량, 리더십

3_1_1_2 :: 분석 기획시 고려사항
 - 1) 가용한 데이터, 2) 적절한 유스케이스, 3) 분석과제 수행 위한 장애요소
 - Variety -- Volume

3_1_2_1 :: 분석 방법론 개요
 - 프레이밍 효과(Framing Effect) :: 문제의 표현 방식에 따라 동일한 사건이나 상황임에도 불구, 
   개인의 판단이나 선택이 달라질 수 있음
 - 암묵지, 형식지, 방법론
 - 나선형 모델 :: 반복을 통해 점증 개발, 처음 적용 용이하나 반복에 대한 관리 체계를 효과적으로 갖추지 
                  못한 경우 복잡도 상승, 프로젝트 진행 난해
 - 단계 / 태스크 / 스텝(WBS, 입력자료, 처리및도구, 출력자료)

3_1_2_2 :: KDD 분석 방법론
 - Fayyad
 - 데이터셋 선택 => 데이터 전처리 => 데이터 변환 => 데이터마이닝 => 데이터 마이닝 결과 평가

3_1_2_3 :: CRISP-DM 분석 방법론
 - 단계 / 일반화 태스크 / 세분화 태스크 / 실행
 - 업무이해 :: 업무목적 파악, 상황파악, 데이터마이닝 목표 설정 / 프로젝트 계획 수립
 - 데이터이해 :: 
 - 데이터 준비 ::
 - 모델링 :: 모델평가(들어있음)
 - 평가 :: 분석결과 평가, 모델링 과정 평가, 모델 적용성 평가
 - 전개 :: 전개계획수립, 모니터링과 유지보수계획수립, 프로젝트 종료보고서작성, 프로젝트리뷰

3_1_2_4 :: 빅데이터 분석 방법론
 - 각 단계는 기준선으로 설정되어 관리, 버전관리 등 통한 통제 필요
 - 분석기획 => 데이터준비 => 데이터분석 => 시스템구현 => 평가및전개

3_1_2_5 :: 분석계획

 - 비즈니스 이해 및 범위 설정 

  - 비즈니스 이해 
    - 입력 :: 업무 메뉴얼, 업무전문가 지식, 빅데이터 분석대상 도메인 관련 자료
    - 출력 :: 비즈니스 이해 및 도메인 문제점

  - 프로젝트 범위 설정 
    - 입력 :: 중장기 계획서, 빅데이터 분석 프로젝트 지시서, 비즈니스 이해 및 도메인 문제점
    - 출력 :: 프로젝트 범위 정의서(SOW)

 - 프로젝트 정의 및 계획 수립
  - 데이터 분석 프로젝트 정의
    - 입력 :: 프로젝트 범위 정의서, 빅데이터 분석 프로젝트 지시서
    - 출력 :: 프로젝트 정의서, 모델 운영 이미지 설계서, 모델 평가 기준

  - 프로젝트 수행 계획 수립
    - 입력 :: 프로젝트 정의서, 모델 운영 이미지 설계서, 모델 평가 기준
    - 출력 :: 프로젝트 수행 계획서, WBS

 - 프로젝트 위험계획 수립
  - 데이터 분석 위험 식별
    - 입력 :: 프로젝트 정의서, 프로젝트 수행 계획서, 선행 프로젝트 산출물 및 정리자료
    - 출력 :: 식별된 위험 목록

  - 위험 대응 계획 수립
    - 입력 :: 식별된 위험 목록, 프로젝트 정의서, 프로젝트 수행 계획서
    - 출력 :: 위험관리계획서

3_1_2_6 :: 데이터 준비

 - 필요 데이터 정의

  - 데이터 정의
    - 입력 :: 프로젝트 수행 계획서, 시스템 설계서, ERD, 메타데이터 정의서, 문서자료
    - 출력 :: 데이터 정의서

  - 데이터 획득 방안 수립
    - 입력 :: 데이터 정의서, 시스템 설계서, ERD, 메타데이터 정의서, 문서자료, 데이터 구입
    - 출력 :: 데이터 획득 계획서

  - 데이터 스토어 설계
   - 정형 데이터 스토어 설계 
    - 입력 :: 데이터정의서, 데이터 획득 계획서
    - 출력 :: 정령 데이터 스토어 설계서, 데이터 매핑 정의서

   - 비정형 데이터 스토어 설계
    - 입력 :: 데이터 정의서, 데이터 획득 계획서
    - 출력 :: 비정형 데이터 스토어 설계서, 데이터 매핑 정의서

 - 데이터 수집 및 정합성 점검

  - 데이터 수집 및 저장
   - 입력 :: 데이터 정의서, 데이터 획득 계획서, 데이터 스토어 설계서
   - 출력 :: 수집된 분석용 데이터

  - 데이터 정합성 점검
   - 입력 :: 수집된 분석용 데이터
   - 출력 :: 데이터 정합성 점검 보고서


3_1_2_7 :: 데이터 분석

 - 분석용 데이터 준비

  - 비즈니스 룰 확인
   - 입력 :: 프로젝트 정의서, 프로젝트 수행 계획서, 데이터 정의서, 데이터 스토어
   - 출력 :: 비즈니스 룰, 분석에 필요한 데이터 범위

  - 분석용 데이터셋 준비
   - 입력 :: 데이터 정의서, 데이터 스토어
   - 출력 :: 분석용 데이터셋

 - 텍스트 분석

  - 텍스트 데이터 확인 및 추출
   - 입력 :: 비정형 데이터 스토어
   - 출력 :: 분석용 텍스트 데이터

  - 텍스트 데이터 분석
   - 입력 :: 분석용 텍스트 데이터, 용어사전
   - 출력 :: 텍스트 분석 보고서

  - 탐색적 분석
   - 입력 :: 분석용 데이터셋
   - 출력 :: 데이터 탐색 보고서

  - 데이터 시각화
   - 입력 :: 분석용 데이터셋
   - 출력 :: 데이터 시각화 보고서

 - 모델링

  - 데이터 분할
   - 입력 :: 분석용 데이터셋
   - 출력 :: 훈련용 데이터, 테스트용 데이터

  - 데이터 모델링
   - 입력 :: 분석용 데이터셋
   - 출력 :: 모델링 결과 보고서

  - 모델 적용 및 운영 방안
   - 입력 :: 모델링 결과 보고서
   - 출력 :: 알고리즘 설명서, 모니터링 방안

 - 모델 평가 및 검증

  - 모델평가
   - 입력 :: 모델링 결과 보고서, 평가용 데이터
   - 출력 :: 모델 평가 보고서

  - 모델검증
   - 입력 :: 모델링 결과 보고서, 모델 평가 보고서, 검증용 데이터
   - 출력 :: 모델 검증 보고서

3_1_2_8 :: 시스템 구현

 - 설계 및 구현

  - 시스템 분석 및 설계
   - 입력 :: 알고리즘 설명서, 운영중 시스템 설계서
   - 출력 :: 시스템 분석 및 설계서

  - 시스템 구현
   - 입력 :: 시스템 분석 및 설계서, 알고리즘 설명서
   - 출력 :: 구현 시스템

 - 시스템 테스트 및 운영

  - 시스템 테스트
   - 입력 :: 구현 시스템, 시스템 테스트 계획서
   - 출력 :: 시스템 테스트 결과보고서

  - 시스템 운영 계획
   - 입력 :: 시스템 분석 및 설계서, 구현 시스템
   - 출력 :: 운영자 매뉴얼, 사용자 매뉴얼, 시스템 운영 계획서
 
     
3_1_2_9 :: 평가 및 전개

   - 입력 :: 구현시스템, 프로젝트 산출물

 - 프로젝트 종료
   - 입력 :: 프로젝트 산출물, 품질관리 산출물, 프로젝트 정의서, 프로젝트 수행 계획서, 프로젝트 성과 평가서
   - 출력 :: 프로젝트 평가 보고서

3_1_3_1 :: 하향식 접근법

 - 문제탐색단계
  - 업무 / 제품 / 고객 / 규제와 감사 / 지원 인프라
  - Operation / Product / Customer / Regulation & Audit / IT & HR
  - 문제탐색은 OPCRI

  - 메가트랜드 --> STEEP :: Social, Technology, Economic, Environmental, Political
  - 경쟁자 확대 --> SCN :: Substitute, Competitor, New_Entrant
  - 시장 니즈 탐색 --> C2I :: Customer, Channel, Influencer
  - 역량 변화 --> IP :: Interal Competency, Partners & Network

  - 외부 참조 모델 기반 문제 탐색
   - 산업별, 업무서비스별 분석테마 후보 그룹(Pool)
   - 분석 유즈 케이스 정의 
 
  - 문제 정의 단계
   - 고객 이탈 문제 :: 고객의 이탈에 영향을 미치는 요인을 식별하고 이탈 가능성을 예측
   - 설비장애 및 판매량 감소 :: 설비 장애를 이끈 신호를 감지, 설비장애요인 식별하고
                                장애 발생 시점 및 가능성 예측
   - 재고관리 및 적정가격 판매 어려움 :: 내부 판매 정보 외 수요 예측 수행할 수 있는 인자 추출 및
                                         모델링 통한 수요 예측

 - 해결방안 탐색 단계
  - 분석기법 및 시스템 --- 분석역량

 - 타당성 검토 단계
  - 경제적 . 데이터 및 기술적 타당성

3_1_3_2 :: 상향식 접근법
 
 - 기존 하향식 접근법의 한계를 극복하기 위한 분석 방법론
 - What 관점
 - EDPIT :: Empathize, Define, Ideate, Prototype, Test
 - 인과관계 --> 상관관계
 - 시행착오를 통한 문제 해결(프로토 타이핑) :: 가설의 생성, 디자인 실험, 실제 환경 테스트, 테스트 결과 통찰, 가설 확인
 - 문제에 대한 인식 수준, 필요 데이터 존재 여부의 불확실성, 데이터 사용 목적의 가변성
 - Data Size / Data Complexity / Speed / Analytic Complexity / Accuracy & Precision
